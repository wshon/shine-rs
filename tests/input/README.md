# 测试输入文件说明

这个目录包含了用于测试 MP3 编码器的各种输入文件和参考输出文件。

## 文件列表

### 音频源文件
- **`sample-12s.wav`** - 12秒的 WAV 格式音频文件，作为编码测试的标准输入
  - 用于验证编码器的基本功能
  - 提供一致的测试基准

### 参考输出文件
- **`sample-12s-shine.mp3`** - 使用 shine 参考实现编码生成的 MP3 文件
  - 通过命令生成：`.\ref\shine\shineenc.exe tests\input\sample-12s.wav tests\input\sample-12s-shine.mp3`
  - 作为我们 Rust 实现的对比基准
  - 用于验证编码输出的正确性

### 标准 MP3 测试文件
- **`sample-12s.mp3`** - 从网络下载的标准 MP3 文件
  - 用于解码测试和格式验证
  - 作为标准 MP3 格式的参考样本

- **`test.mp3`** - 从网络下载的标准 MP3 文件
  - 用于各种 MP3 解析和验证测试
  - 提供不同的音频内容和编码参数测试场景

## 使用方式

### 生成参考文件
```bash
# 使用 shine 参考实现生成标准输出
.\ref\shine\shineenc.exe tests\input\sample-12s.wav tests\input\sample-12s-shine.mp3
```

### 测试流程
1. **编码测试**：使用 `sample-12s.wav` 作为输入，通过我们的 Rust 编码器生成输出
2. **参考对比**：与 `sample-12s-shine.mp3` 进行对比验证编码正确性
3. **格式验证**：使用 `sample-12s.mp3` 和 `test.mp3` 验证 MP3 解析和格式处理
4. **标准兼容性**：确保我们的实现能正确处理标准 MP3 文件

## 测试验证
这些文件主要用于：
- **编码功能验证** - 确保编码器能正确处理 WAV 输入
- **输出对比** - 验证与 shine 参考实现的一致性
- **格式兼容性** - 使用标准 MP3 文件验证解析和处理能力
- **回归测试** - 防止代码修改影响编码质量
- **集成测试** - 验证完整的编码和解析流程

## 注意事项
- 所有测试文件都应该保持版本控制，确保测试的一致性
- 参考输出文件 (`sample-12s-shine.mp3`) 是验证标准，不应随意修改
- 标准 MP3 文件 (`sample-12s.mp3`, `test.mp3`) 用于格式兼容性验证
- 新增测试文件时，应该同时生成对应的 shine 参考输出
